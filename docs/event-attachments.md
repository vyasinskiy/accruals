# Процесс: Прикрепление документов и фото к событиям (Event Attachments & S3 Upload)

Этот процесс описывает отправку фотографий и документов из Telegram-бота или веб-панели управления, их автоматическую загрузку в S3-хранилище и привязку к запланированным событиям (`ScheduledEvent`).

---

## 1. Сценарий пересылки фото из Telegram-бота

### Шаг 1. Получение фотографии от администратора
1. Администратор (или суперадмин `SUPER_ADMIN_TELEGRAM_ID`) отправляет или пересылает в Telegram-бот фото (например, снимки счетчиков или чеки, присланные арендатором в мессенджере).
2. Бот определяет изображение и предлагает интерактивное меню действий:
   - `📌 Прикрепить к событию` (`attach_photo_start`)
   - `❌ Отмена` (`attach_photo_cancel`)

### Шаг 2. Выбор фильтра событий
При нажатии кнопки `📌 Прикрепить к событию` бот запрашивает выбор категории фильтра:
- **👤 По арендатору** (`attach_filter_tenant`): выводится список всех арендаторов. При выборе арендатора загружаются относящиеся к нему запланированные события.
- **🏢 По квартире** (`attach_filter_apartment`): выводится список квартир.
- **📑 По лицевому счету** (`attach_filter_account`): выводится список лицевых счетов с указанием адреса квартиры.
- **🌐 Все активные события** (`attach_filter_all`): выводится полный список активных событий.

### Шаг 3. Загрузка файла в S3 и сохранение в базе данных
1. Бот запрашивает ссылку на скачивание файла через Telegram Bot API (`ctx.telegram.getFileLink`).
2. Загружает бинарный буфер файла и передает его в микросервис `accountant` через брокер сообщений RabbitMQ (`attach_event_document`).
3. Сервис `accountant` генерирует уникальный S3 ключ формата `events/<eventId>/attachments/<timestamp>-<filename>` и загружает файл в S3 хранилище (`S3StorageService.uploadBuffer`).
4. В PostgreSQL создается запись в таблице `event_attachments`:
   - `scheduled_event_id`: ID события
   - `file_name`: имя файла
   - `s3_key`: путь в S3
   - `mime_type`: MIME-тип (например `image/jpeg`)
   - `uploaded_by`: имя администратора Telegram
5. Бот отправляет сообщение о высокой успешности операции и загрузке в S3.

---

## 2. Отображение и управление в панели администратора (Admin UI)

1. На странице детализации события `/events/[id]` создан блок **"Прикрепленные документы и фотографии"**.
2. В блоке отображается галерея загруженных файлов:
   - Для изображений создается миниатюра с возможностью открытия на весь экран.
   - Для документов отображается иконка прикрепленного файла.
   - Указываются имя файла, дата и время загрузки, а также источника (`Telegram` или `Admin UI`).
3. При нажатии на **"Скачать"** генерируется временная подписанная ссылка S3 (`getSignedDownloadUrl`).
4. В блоке также доступна кнопка прямой ручной загрузки файла через веб-интерфейс, а также удаление документа из базы данных и S3.
