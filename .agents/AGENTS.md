# Правила и инструкции проекта для ассистента Antigravity

Этот репозиторий следует определенным соглашениям по базам данных, стилю кода и структуре процессов. Руководствуйтесь следующими правилами при внесении изменений или расширении кодовой базы.

## 1. Бизнес-процессы и сценарии
Подробная документация основных процессов расположена в папке [docs/](file:///Users/usuario/code/accruals/docs/):
- [meter-submission.md](file:///Users/usuario/code/accruals/docs/meter-submission.md): Процесс ежемесячной подачи показаний приборов учета (счетчиков), оповещения на основе индивидуального дня отправки для счета (по умолчанию 20-го числа каждого месяца), повторные напоминания по понедельникам и кнопки подтверждения.
- [latest-invoices.md](file:///Users/usuario/code/accruals/docs/latest-invoices.md): Просмотр последних начислений, пагинация в Telegram, вертикальная раскладка кнопок скачивания инвойсов из S3 и парсинг сырого JSON.

**Важное правило**: При любых изменениях или расширениях бизнес-процессов, соответствующие файлы описания процессов в папке `docs/` должны быть в обязательном порядке обновлены и приведены в соответствие с изменениями.

---

## 2. Соглашения по форматированию интерфейса Telegram
- **Сумма и статус начислений**: Запрещено выводить сырые технические строки разметки из БД (например, `initialBalance=...`). Вместо этого парсите поле `item.rawJson` в сервисе бота, выводя чистое отформатированное значение суммы (например, `1048.33 руб.`) и человекочитаемые русские статусы (например, `Оплачено`, `Ожидает оплаты`).

---

## 3. Правила написания кода и типизации
- **Запрет на использование `any`**: Использование типа `any` в TypeScript коде строго запрещено. Вместо этого используйте строгие типы, интерфейсы или тип `unknown` с последующим сужением типов (type guarding).
