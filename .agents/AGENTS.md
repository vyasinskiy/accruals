# Правила и инструкции проекта для Antigravity

Этот репозиторий следует определенным паттернам проектирования, соглашениям по базам данных и структуре процессов. Руководствуйтесь следующими правилами при внесении изменений или расширении кодовой базы.

**ВАЖНО**: Для проекта созданы файлы описания бизнес-процессов в папке [docs/](file:///Users/usuario/code/accruals/docs/). Любые изменения в бизнес-логике или шагах процессов должны быть в обязательном порядке отражены в соответствующих файлах документации (например, при доработках логики счетчиков должен обновляться [meter-submission.md](file:///Users/usuario/code/accruals/docs/meter-submission.md)).

## 1. Архитектурное разделение (Decoupling)
- **Принцип**: Основная бизнес-логика должна быть полностью изолирована от форматирования UI и деталей доставки уведомлений (таких как HTML/Markdown-теги Telegram-ботов или кастомные кнопки).
- **Сценарий работы**:
  1. Ядро сервиса (например, `accountant`) выполняет расчеты или транзакции в базе данных.
  2. Ядро отправляет структурированное событие (например, `accrual_upserted`, `meter_submission_required`), содержащее только чистые данные, через брокер сообщений RabbitMQ.
  3. Интерфейсный сервис (например, `telegram-bot`) слушает эти события, форматирует локализованные текстовые сообщения для пользователя, прикрепляет интерактивные inline-кнопки и осуществляет доставку.

## 2. Соглашения по базе данных Prisma
- **Изолированные клиенты**: Чтобы предотвратить конфликты типов TypeScript между микросервисами (где генерация клиента для одного сервиса перезаписывает клиент другого), в проекте используются локально изолированные клиенты.
  - Файлы `schema.prisma` явно выводят клиент в локальные папки: `output = "../src/generated/client"`.
  - Сервисы импортируют свой конкретный клиент напрямую: `import { PrismaClient } from '../../generated/client'`.
- **Версионирование и запуск CLI**:
  - Всегда используйте скрипты из `package.json` (например, `npm run prisma:generate`, `npm run prisma:migrate`) вместо прямых вызовов `npx prisma`, чтобы жестко зафиксировать версию Prisma на `6.6.0` и избежать ломающих изменений.
  - Для проведения миграций схемы на этапе разработки запускайте:
    ```bash
    DATABASE_URL="postgresql://postgres:postgres@localhost:5432/accountant_db" DIRECT_URL="postgresql://postgres:postgres@localhost:5432/accountant_db" npm run prisma:migrate-dev --workspace=apps/accountant -- --name <migration_name>
    ```
    *(Необходимо переопределять обе переменные DATABASE_URL и DIRECT_URL в командной строке, чтобы направить миграцию на правильный контейнер БД и избежать расхождения схем).*

## 4. Соглашения по форматированию интерфейса Telegram
- **Сумма и статус начислений**: Запрещено выводить сырые технические строки разметки из БД (например, `initialBalance=...`). Вместо этого парсите поле `item.rawJson` в сервисе бота, выводя чистое отформатированное значение суммы (например, `1048.33 руб.`) и человекочитаемые русские статусы (например, `Оплачено`, `Ожидает оплаты`).

## 5. Правила написания кода и типизации
- **Запрет на использование `any`**: Использование типа `any` в TypeScript коде строго запрещено. Вместо этого используйте строгие типы, интерфейсы или тип `unknown` с последующим сужением типов (type guarding).
